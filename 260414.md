## Übung (Anzahl Einträge in einem Verzeichnis)

Schreibe ein shellscript das als Argument einen Pfad auf ein Verzeichnis erhält. Das Script soll die Anzahl der Einträge in diesem Verzeichnis als Zahl ausgeben. Dies soll aus der `ls` Ausgabe mit `wc` ermittelt werden.

```bash
$ ./nbrentries.sh ~/test
Es sind 12 Einträge im dir ~/test
```

Hinweis - Vergleiche die Ausgaben von

```bash
ls
ls | cat
echo "$(ls)"
```

Dokumentiere den Unterschied, was ist die Logik dahinter.

**nbrentries.sh**
```bash
#!/bin/env bash

path="$1"

echo "Number of entries: $(ls "$path" | wc -l)"
```

**Unterschied zwischen ```ls```, ```ls | cat``` und ```echo $(ls)```**

| Befehl    | Merkmall | 
| ------- | -------- | 
| ```ls``` | Ergibt eine Liste der Einträge vom Verzeichniss in eine Reihe |
| ```ls ǀ cat``` | Ergibt die Liste der Einträge vom Verzeichniss in einer Tabelle (jeder Eintrag hat eigene Reihe)    |
| ```echo $(ls)``` | Ergibt fasst dieselbe Ausgabe wie ```ls```. Es unterscheidet sich, dass die Ausgabe eine farblose Textausgabe ist |

## Übung (Tage bis zum Ball)

Schreibe ein Shell-Skript das die Anzahl der Tage bis zum HTL Ball ermittelt. Die Ausgabe soll in der folgenden Form sein.

```
Es sind noch 42 Tage bis zum HTL Ball (2027-01-16)
```

Hinweise:

- Verwende das `date` Format für  `seconds since the Epoch (1970-01-01 00:00 UTC)`

**Skript**
```bash
#!/bin/env bash

curentime=$(date +%s)
balltime=$(date -d "2027-01-16" +%s)

diff=$(($balltime - $curentime))

echo "Es sind noch $(($diff / 86400)) Tage bis zum HTL Ball ($(date -d @$balltime +"%Y-%m-%d"))!"
```

## Übung (Zufälliger Satz)

Erstelle ein Skript das einen Satz aus 5 zufälligen Wörtern bildet, z.B.:

```
richtig ganzen Kilometer auf Mittel
```

Wähle die Wörter aus dieser [Wortliste](https://franzmatejka.at/htl/doc/SYTB_3/testdata/wortliste1000.txt).

- Hinweis: Das Komando `sed -n 5p datei.txt` gibt die 5te Zeile aus. 
- Erweiterung: Das Skript soll auch die Anzahl der in der Wortliste enthaltenen Wörter ermitteln. D.h. die Konstante 1000 soll im Skript nicht vorkommen.

**Skript**
```bash
#!/bin/env bash

words=$(cat wortliste1000.txt | wc -l)

sentence=""

for i in {1..5}; do
  random_number=$((RANDOM % words + 1))
  random_word=$(sed -n "${random_number}p" wortliste1000.txt)
  sentence+="$random_word "
done

echo "$sentence"
```

## Übung (dated copy V1)

Create a script which will take a filename as its first argument and create a dated copy of the file. eg. if our file was named `file1.txt` and today is the 2021-10-29 it would create a copy such as `2021-10-29_file1.txt`. (To achieve this you will probably want to play with command substitution and the command `date`)

**Script**
```bash
#!/bin/env bash

filename="$1"

today=$(date +"%Y-%m-%d")

cp "$filename" "$today"_"$filename"
```


## Übung (dated copy V2)

**Challenge:** To make it a bit harder, see if you can get it so that the date is after the name of the file (eg. `file1_2021-10-29.txt`) (Hint: Use `cut` to seperate filename from extension, like: `echo "filename.ext" | cut ...`)

```bash
#!/bin/env bash

filename_with_extension="$1"
filename="$(echo $filename_with_extension | cut -d. -f1)"
extension="$(echo $filename_with_extension | cut -d. -f2)"

today=$(date +"%Y-%m-%d")

cp "$filename_with_extension" "$filename"_"$today"."$extension"
```

## Übung (dated copy V3)

**Challenge:** Now see if you can expand the previous question to accept a list of files on the command line and it will create a named copy of all of them. (Hint: The command [xargs](https://ryanstutorials.net/linuxtutorial/bonus.php#xargs) may be useful here.)

```bash
#!/bin/env bash

files="$@"
today=$(date +"%Y-%m-%d")

for f in $files
do
    filename_with_extension="$f"
    filename="$(echo $filename_with_extension | cut -d. -f1)"
    extension="$(echo $filename_with_extension | cut -d. -f2)"
    cp "$filename_with_extension" "$filename"_"$today"."$extension"
done
```
