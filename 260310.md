Ich habe in vorherigen Beispiel ein Reasponse mit dem Status Code 200 erhaltet. Code 200 steht für "OK". Damit wird eine Erfolgreiche Verbindung auf dem Server bezeichnet.

**5) Geschlechtsschätzung anhand von Namen. Lade mit curl → du bekommst eine Fehlermeldung. Ein Parameter wird an die URL so angefügt: ?name=value. Ermittle das wahrscheinliche Geschlecht von: Noor, Ariel, Amina, Elowen, Levin.**

- Hinweis: verwende Quotes ("\<URL\>") in der curl Kommandozeile, da ? in der shell eine spezielle Bedeutung hat.
  Das Datenformat, das hier vom Server gesendet wird, ist JSON. Ganz viele Web-APIs verwenden JSON. Verwende die Option zum Anzeigen des Response-Headers und suche darin nach einem Hinweis auf dieses Datenformat.

Mit https://api.genderize.io kann man einen Namen eingeben, um den möglichen Geschlecht herauszufinden. Man muss nur eine Querry-Parameter "name" dazu geben.

**Befehle**

```bash
$ curl "https://api.genderize.io?name=Noor"
```

**Reaspons**

```json
{
  "count": 167923,
  "name": "Noor",
  "gender": "female",
  "probability": 0.61
}
```

_Die ermitellte Genders:_

| name   | gender |
| ------ | ------ |
| Noor   | female |
| Ariel  | male   |
| Amina  | female |
| Elowen | female |
| Levin  | male   |

Mit der Option -i kann man die Datentyp finden. Er steht in Header **content-type**.

**Bash**

